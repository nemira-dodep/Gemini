# Gemini CLI 🚀

CLI приложение для работы с Google Gemini API. Позволяет взаимодействовать с различными моделями Google Gemini через командную строку.

## Возможности

✨ **Множество моделей:**
- **gemini-3-flash** — основная модель с новейшими знаниями 2026
- **gemini-3.1-flash-lite** — высокие лимиты (500 запросов/день)
- **gemma-4-31b-it** — огромные лимиты для массовой обработки
- **deep-research-pro** — глубокий поиск и создание документации
- **gemini-3.1-flash-image** — анализ скриншотов и графиков

📊 **Отслеживание использования** — автоматический учет запросов и соблюдение дневных лимитов

🎨 **Красивый вывод** — форматирование Markdown с цветной подсветкой в терминале

⚙️ **Интерактивный режим** — выбор моделей и параметров через меню

## Требования

- Node.js 14+
- Активный Google Gemini API ключ

## Установка

### 1. Клонируйте или скачайте репозиторий
```bash
cd Gemini
```

### 2. Установите зависимости
```bash
npm install
```

### 3. Получите API ключ
1. Откройте [Google AI Studio](https://aistudio.google.com/apikey)
2. Нажмите "Create API key"
3. Скопируйте ключ

### 4. Установите переменную окружения

**Способ 1: Через .env файл (рекомендуется)**
```bash
# Создайте файл .env в корне проекта
echo GEMINI_API_KEY=ваш_ключ_здесь > .env
```

**Способ 2: Через переменную окружения (Windows PowerShell)**
```powershell
$env:GEMINI_API_KEY = "ваш_ключ_здесь"
```

**Способ 3: Через переменную окружения (Windows CMD)**
```cmd
set GEMINI_API_KEY=ваш_ключ_здесь
```

**Способ 4: Через переменную окружения (Linux/Mac)**
```bash
export GEMINI_API_KEY="ваш_ключ_здесь"
```

## Использование

### Основной режим (cli.js) ⭐ Рекомендуется

```bash
node cli.js "ваш запрос"
```

Или выбрать модель интерактивно:
```bash
node cli.js
```

**Основные команды и флаги:**

| Команда | Описание |
|---------|---------|
| `node cli.js "запрос"` | Просто вопрос без создания файлов |
| `node cli.js "запрос" --save` | Вопрос + сохранение файлов (если модель их предложит) |
| `node cli.js --scan` | Проверить доступные модели Google |
| `node cli.js --stats` | Статистика всех моделей за день |
| `node cli.js --stats flash` | Статистика конкретной модели |
| `node cli.js --help` | Полная справка |

**Флаги:**

| Флаг | Описание |
|------|---------|
| `-m <model>` | Выбрать модель (flash, lite, gemma, research, vision) |
| `-c, --config` | Интерактивный выбор модели |
| `--save` | **Создавать файлы из ответов** (по умолчанию отключено) |
| `-d, --dry-run` | Показать какие файлы будут созданы без сохранения |
| `--scan` | Список доступных моделей |
| `--stats [model]` | Показать статистику |
| `-h, --help` | Справка |

### Простой режим (script.js)

```bash
node script.js "ваш запрос здесь"
```

**Примеры:**
```bash
node script.js "Что такое REST API?"
node script.js "Как установить Node.js?"
```

## Примеры использования

### 1️⃣ Просто задать вопрос (без создания файлов)

```bash
node cli.js -m flash "Как работает Promise в JavaScript?"
```

✅ Выведет ответ  
❌ Не создаст файлы

### 2️⃣ Просить создать файл (с --save)

```bash
node cli.js -m flash "Создай файл sort.js с функцией быстрой сортировки" --save
```

✅ Выведет ответ  
✅ Создаст файл sort.js

### 3️⃣ Генерация кода

```bash
# Просто вывести код
node cli.js -m flash "Напиши функцию для валидации email"

# Сохранить в файл
node cli.js -m flash "Создай validators.js с функцией валидации email" --save
```

### 4️⃣ Анализ текста (большие объемы)

```bash
node cli.js -m gemma "Проанализируй текст на опечатки и улучши стиль" --save
```
Затем загрузите файл с текстом.

### 5️⃣ Анализ кода на ошибки

```bash
node cli.js -m flash "Проверь код на ошибки" --save
```
Загрузите файл с кодом.

### 6️⃣ Анализ изображений/скриншотов

```bash
node cli.js -m vision "Описи что видишь на скриншоте"
```
Загрузите скриншот.

### 7️⃣ Проверка лимитов

```bash
# Полная статистика за день
node cli.js --stats

# Конкретная модель
node cli.js --stats gemma
node cli.js --stats flash
```

### 8️⃣ Dry-run режим (показать файлы без сохранения)

```bash
node cli.js -m flash "Создай config.json с настройками" -d
```

### 9️⃣ Выбрать модель интерактивно

```bash
node cli.js -c "Напиши Hello World на Python"
```
Появится меню для выбора модели.

### 🔟 Проверить доступные модели

```bash
node cli.js --scan
```

Показывает все доступные модели с лимитами RPM и TPM.

## 📊 Счетчик токенов и статистика

Приложение **автоматически считает реальные токены**, потребленные каждой моделью.

### Как работает счетчик

✅ **Сохраняет:**
- Количество запросов
- Входные + выходные токены
- Примерную стоимость в USD

✅ **Сбрасывает статистику:**
- Ежедневно в 00:00

### Просмотр статистики

**Полная статистика за день:**
```bash
node cli.js --stats
```

**Вывод:**
```

Дата: Tue May 12 2026

flash      │ 5   запросов │ 2450   токенов │ $0.3675
lite       │ 3   запросов │ 1200   токенов │ $0.0900
gemma      │ 12  запросов │ 15230  токенов │ $0.4569
──────────────────────────────────────────────────────
ИТОГО:     │ 20  запросов │ 18880  токенов │ $0.9144
```

**Статистика конкретной модели:**
```bash
node cli.js --stats flash
```

**Вывод:**
```
📊 СТАТИСТИКА ИСПОЛЬЗОВАНИЯ
Дата: Tue May 12 2026

FLASH:
  Запросов:     5
  Токенов:      2450
  Ср. на запрос: 490 токенов
  Стоимость:    ~$0.3675
```

### После каждого запроса

Приложение показывает детальную статистику:
```
──────────────────────────────────────────────
📊 СТАТИСТИКА [flash]:
   Запрос:       245 входных + 89 выходных = 334 токенов
   За день:      2450 токенов (5 запросов)
   Примерная стоимость: ~$0.3675
──────────────────────────────────────────────
```

## Управление лимитами

Каждая модель имеет **дневные лимиты** запросов:

| Модель | Лимит/день | Использование | Цена |
|--------|-----------|------|------|
| flash | 20 | Основная работа, новейшие знания | $0.15/1k |
| lite | 500 | Рутинные задачи, простые вопросы | $0.075/1k |
| gemma | 14400 | Массовая обработка текста | $0.03/1k |
| research | 500 | Глубокий анализ и документация | $0.15/1k |
| vision | 20 | Анализ изображений и скриншотов | $0.15/1k |

### Что делать если лимит исчерпан?

```
❌ [429 Too Many Requests] You exceeded your current quota
```

**Решения:**
1. Используйте модель с выше лимитом:
   ```bash
   node cli.js -m gemma "ваш запрос"  # 14400 запросов/день
   ```

2. Подождите сброса лимита (23:59 UTC)

3. Проверьте текущее использование:
   ```bash
   node cli.js --stats
   ```

4. Обновите план в [Google AI Studio](https://aistudio.google.com)

## Обработка файлов

⚠️ **ВАЖНО:** По умолчанию файлы **НЕ создаются автоматически**!

Это предотвращает случайное создание файлов при обычных запросах.

### ✅ Когда создаются файлы

Файлы создаются ТОЛЬКО когда:
1. Вы используете флаг `--save`
2. И модель предложит текст в формате `<file path="...">содержимое</file>`

### ❌ Когда файлы НЕ создаются

```bash
# НЕ создаст файл - просто выведет ответ
node cli.js -m flash "Напиши функцию для сортировки"

# НЕ создаст файл - просто вывод
node cli.js -m gemma "Создай index.html" 
```

### ✅ Как сохранить результат

```bash
# Создаст файл, если модель его предложит
node cli.js -m flash "Создай файл sort.js с функцией быстрой сортировки" --save

# Создаст файл, если модель его предложит
node cli.js -m gemma "Создай index.html для веб-приложения" --save
```

### 📂 Загрузка файлов на анализ

При любом запросе приложение может загружать файлы:
- Текстовые файлы (`.txt`, `.md`, `.json`, `.js`, `.py`, `.css` и т.д.)
- Изображения (`.jpg`, `.png`, `.gif`, `.webp`)
- Максимальный размер: 20 МБ

**Пример с загрузкой:**
```bash
node cli.js -m gemma "Проанализируй этот код на ошибки" --save
→ Выберите файл для загрузки
→ Модель анализирует и может создать файл с исправлениями
```

### 🔍 Примеры реальных сценариев

**Сценарий 1: Анализ без сохранения**
```bash
node cli.js -m flash "Найди ошибки в коде"
# Загружаете файл
# Получаете ответ и анализ
# Файлы НЕ создаются
```

**Сценарий 2: Анализ с автоисправлением**
```bash
node cli.js -m flash "Найди ошибки и создай исправленный файл" --save
# Загружаете файл
# Модель создает исправленный файл
```

**Сценарий 3: Dry-run (показать без сохранения)**
```bash
node cli.js -m flash "Создай конфиг-файл" -d
# Видите какой файл будет создан
# Файл НЕ сохраняется на диск
```

### 🎛️ Флаги для работы с файлами

| Флаг | Поведение |
|------|-----------|
| `(без флагов)` | Выводит ответ, файлы НЕ создаются |
| `--save` | Создает файлы если модель их предложит |
| `-d, --dry-run` | Показывает какие файлы будут созданы (без сохранения) |

## Решение проблем

### Ошибка: "GEMINI_API_KEY не найдена"

**Решение:** 
1. Проверьте, что файл `.env` существует в корне проекта
2. Убедитесь, что ключ скопирован правильно (без пробелов)
3. Перезагрузите терминал после добавления переменной

### Ошибка: "HTTP 429 - Too Many Requests"

**Решение:** Вы превысили дневной лимит для этой модели. Используйте другую модель или дождитесь следующего дня.

### Ошибка: "Module not found"

**Решение:** Переустановите зависимости:
```bash
npm install
```

### Нет вывода после запроса

**Решение:** 
- Дождитесь ответа (запрос может занять время)
- Проверьте интернет соединение
- Проверьте статус Google API в консоли
