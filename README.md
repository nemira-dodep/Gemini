# Gemini CLI 🚀

CLI приложение для работы с Google Gemini API. Позволяет взаимодействовать с различными моделями Google Gemini через командную строку.

## Возможности

✨ **Множество моделей:**
- **gemini-3-flash** — основная модель с новейшими знаниями 2026
- **gemini-3.1-flash-lite** — высокие лимиты (500 запросов/день)
- **gemma-4-31b-it** — огромные лимиты для массовой обработки
- **deep-research-pro** — глубокий поиск и создание документации
- **gemini-3.1-flash-image** — анализ скриншотов и графиков

📊 **Отслеживание использования** — автоматический учет запросов и соблюдение дневных лимитов

🎨 **Красивый вывод** — форматирование Markdown с цветной подсветкой в терминале

⚙️ **Интерактивный режим по умолчанию** — запуск `gemini` открывает чат с домашним экраном, slash-командами и настройками

⚡ **Потоковый вывод (Streaming)** — мгновенная печать ответа по мере генерации (как в веб-версии)

🖼️ **Поддержка любых файлов** — автоматическое определение форматов (текст, PDF, JPG, PNG, WebP)

## Требования

- Node.js 20+
- Активный Google Gemini API ключ

## Установка

### 1. Клонируйте или скачайте репозиторий
```bash
cd Gemini
```

### 2. Установите зависимости
```bash
npm install
```

### 3. Получите API ключ
1. Откройте [Google AI Studio](https://aistudio.google.com/apikey)
2. Нажмите "Create API key"
3. Скопируйте ключ

### 4. Глобальная установка (Рекомендуется) 🌟

Чтобы запускать утилиту из любой папки на вашем компьютере, просто выполните в папке проекта:
```bash
npm link
# или
npm install -g .
```
Теперь вместо длинного `gemini` вы можете просто использовать глобальную команду `gemini`!

### 5. Сохраните API ключ (Два способа)

**Способ 1: Встроенной командой (Рекомендуется)**
```bash
gemini --set-key "ваш_ключ_здесь"
```

**Способ 2: Через переменную окружения GEMINI_API_KEY**
Если вы не хотите использовать встроенный конфиг, вы можете добавить ключ в файл `.env` в корне проекта или задать системную переменную окружения ОС.

## Использование

### Основной режим (Глобальная команда) ⭐ Рекомендуется

```bash
gemini
```

Запуск без аргументов открывает интерактивный чат. Для одноразового запроса по-прежнему можно передать текст сразу:
```bash
gemini "ваш запрос"
```

Выбрать модель по умолчанию:
```bash
gemini -c
```

**Основные команды и флаги:**

| Команда | Описание |
|---------|---------|
| `gemini "запрос"` | Просто вопрос без создания файлов |
| `gemini "запрос" --save` | Вопрос + сохранение файлов (если модель их предложит) |
| `gemini` | Интерактивный чат с домашним экраном и slash-командами |
| `gemini -i "запрос" file.js` | Интерактивный чат со стартовым сообщением и файлами |
| `gemini --scan` | Проверить доступные модели Google |
| `gemini --sync-models` | Сохранить модели, реально доступные по вашему API ключу |
| `gemini --models` | Показать локально настроенные модели без запроса к API |
| `gemini --stats` | Статистика всех моделей за день |
| `gemini --stats flash` | Статистика конкретной модели |
| `gemini --reset-stats` | Сбросить локальную статистику |
| `gemini --config-path` | Показать пути к конфигу и статистике |
| `gemini --doctor` | Проверить окружение, ключ, конфиг и зависимости |
| `gemini --help` | Полная справка |

**Флаги:**

| Флаг | Описание |
|------|---------|
| `--set-key` | Сохранить API-ключ в конфигурации (вместо `.env`) |
| `-m <model>` | Использовать модель только для этого запроса (не сохраняя) |
| `-c, --config` | Выбрать и сохранить модель по умолчанию |
| `-i, --interactive` | Запустить режим непрерывного чата (с историей сообщений) |
| `--save` | **Создавать файлы из ответов** (по умолчанию отключено) |
| `-d, --dry-run` | Показать какие файлы будут созданы без сохранения |
| `--scan` | Список доступных моделей |
| `--sync-models` | Обновить локальный список моделей из Google API |
| `--models` | Локальный список настроенных моделей |
| `--stats [model]` | Показать статистику |
| `--reset-stats` | Сбросить локальную статистику |
| `--config-path` | Показать расположение `config.json` и `usage.json` |
| `--settings` | Открыть интерактивные настройки |
| `--doctor` | Диагностика локальной установки |
| `-h, --help` | Справка |

### Простой режим (script.js)


## Примеры использования

### 💬 Режим чата (Conversational Mode)

Если вы хотите задать несколько связанных вопросов (чтобы модель помнила контекст предыдущих сообщений), просто запустите CLI без аргументов:

```bash
gemini
```
Вы также можете загрузить файлы в чат при его запуске:
```bash
gemini -i "Объясни этот код" index.js
```

Внутри чата доступны команды:

| Команда | Что делает |
|---------|------------|
| `/model` или `/m` | Выбрать модель для текущей сессии |
| `/model lite` | Быстро переключиться на конкретную модель |
| `/settings` или `/config` | Открыть настройки CLI |
| `/stats` или `/stats flash` | Показать статистику |
| `/models` | Показать локально настроенные модели |
| `/models --refresh` | Обновить и показать модели |
| `/sync-models` | Синхронизировать модели по API ключу |
| `/doctor` | Проверить окружение |
| `/save on/off` | Включить или выключить сохранение файлов |
| `/dry on/off` | Включить или выключить dry-run |
| `/search on/off` | Включить или выключить Google Search |
| `/files README.md "*.js"` | Добавить файлы к следующему сообщению |
| `/clear` или `/new` | Очистить историю чата |
| `/home` | Показать домашний экран |
| `/exit` или `/q` | Выйти |

В чате также работают привычные CLI-команды с префиксом `gemini`: например `gemini --sync-models`, `gemini --models`, `gemini --doctor`, `gemini -m lite`.

### 1️⃣ Просто задать вопрос (без создания файлов)


✅ Выведет ответ  
❌ Не создаст файлы

✅ Выведет ответ  
✅ Создаст файл sort.js

```bash
# Сохранить в файл

```

### 4️⃣ Анализ текста (большие объемы)

```bash
gemini -m gemma "Проанализируй текст на опечатки и улучши стиль" --save
```
Затем загрузите файл с текстом.

### 5️⃣ Анализ кода на ошибки

```bash
gemini -m flash "Проверь код на ошибки" --save
```
Загрузите файл с кодом.

### 6️⃣ Анализ изображений/скриншотов

CLI автоматически определяет формат изображения (JPG, PNG, WebP) и отправляет его модели с правильным MIME-типом.

```bash

```

### 7️⃣ Проверка лимитов

```bash
# Полная статистика за день
gemini --stats

# Конкретная модель
gemini --stats gemma
gemini --stats flash
```

### 7️⃣.1️⃣ Сервисные команды

```bash
# Локально настроенные модели
gemini --models

# Где лежат конфиг и статистика
gemini --config-path

# Открыть настройки
gemini --settings

# Сбросить статистику за день
gemini --reset-stats
```

### 8️⃣ Dry-run режим (показать файлы без сохранения)

```bash
gemini -m flash "Создай config.json с настройками" -d
```

### 9️⃣ Выбрать модель по умолчанию

```bash
gemini -c
```
Появится меню для выбора. Выбранная модель сохранится в пользовательском `config.json` и будет использоваться для всех последующих запросов, если не переопределена флагом `-m`.

### 🔟 Проверить доступные модели

```bash
gemini --scan
```

Показывает все доступные модели с лимитами RPM и TPM.

Чтобы CLI использовал только модели, реально доступные вашему API ключу, синхронизируйте список:

```bash
gemini --sync-models
```

Команда получает список моделей из Google API и дополнительно проверяет каждую модель через лёгкий `countTokens` запрос. Модели, которые дают `403 Forbidden` или другую ошибку доступа, скрываются из `/model`.

После этого `gemini --models` и `/model` внутри чата будут показывать сохранённый список. В интерактивном режиме CLI также обновляет модели автоматически, если кеш старше суток или ещё пустой.

### 1️⃣1️⃣ Проверить локальную установку

```bash
gemini --doctor
```

Команда проверяет версию Node.js, наличие API-ключа, модель по умолчанию, статистику и основные зависимости.

По умолчанию конфиг и статистика теперь хранятся вне папки проекта:

- Windows: `%USERPROFILE%\.gemini-cli\config.json` и `usage.json`
- macOS/Linux: `~/.gemini-cli/config.json` и `usage.json`

Можно переопределить папку переменной окружения `GEMINI_CLI_HOME`. Старые файлы `gemini-config.json` и `gemini-usage.json` в папке проекта всё ещё читаются для совместимости, но новые записи идут в пользовательскую папку.

## 📊 Счетчик токенов и статистика

Приложение **автоматически считает реальные токены**, потребленные каждой моделью.

### Как работает счетчик

✅ **Сохраняет:**
- Количество запросов
- Входные + выходные токены
- Примерную стоимость в USD

✅ **Сбрасывает статистику:**
- Ежедневно в 00:00

## Управление лимитами

Каждая модель имеет **дневные лимиты** запросов:

| Модель | Лимит/день | Использование | Цена |
|--------|-----------|------|------|
| flash | 20 | Основная работа, новейшие знания | $0.15/1k |
| lite | 500 | Рутинные задачи, простые вопросы | $0.075/1k |
| gemma | 14400 | Массовая обработка текста | $0.03/1k |
| research | 500 | Глубокий анализ и документация | $0.15/1k |
| vision | 20 | Анализ изображений и скриншотов | $0.15/1k |

### Что делать если лимит исчерпан?

```
❌ [429 Too Many Requests] You exceeded your current quota
```

**Решения:**
1. Используйте модель с выше лимитом:
   ```bash
   gemini -m gemma "ваш запрос"  # 14400 запросов/день
   ```

2. Подождите сброса лимита (23:59 UTC)

3. Проверьте текущее использование:
   ```bash
   gemini --stats
   ```

4. Обновите план в [Google AI Studio](https://aistudio.google.com)

## Обработка файлов

⚠️ **ВАЖНО:** По умолчанию файлы **НЕ создаются автоматически**!

Это предотвращает случайное создание файлов при обычных запросах.

### ✅ Когда создаются файлы

Файлы создаются ТОЛЬКО когда:
1. Вы используете флаг `--save`
2. И модель предложит текст в формате `<file path="...">содержимое</file>`

### ❌ Когда файлы НЕ создаются

```bash
# НЕ создаст файл - просто выведет ответ
gemini -m flash "Напиши функцию для сортировки"

# НЕ создаст файл - просто вывод
gemini -m gemma "Создай index.html" 
```

### ✅ Как сохранить результат

При использовании флага `--save` или `-d` приложение переходит в режим **Structured Outputs**. Это гарантирует, что файлы будут созданы со 100% надежностью благодаря JSON-формату.

> [!IMPORTANT]
> В режиме `--save` или `-d` (dry-run) потоковый вывод (стриминг) временно отключается. Это необходимо, чтобы приложение могло получить полный JSON-объект от модели и корректно распарсить его.

```bash
# Создаст файл, если модель его предложит (через JSON Schema)
gemini -m flash "Создай файл sort.js с функцией быстрой сортировки" --save
```

### 📂 Загрузка файлов на анализ

При любом запросе приложение может загружать файлы. Поддерживаются **Glob-маски** (шаблоны поиска), что позволяет удобно загружать целые папки.

Файлы можно указывать до или после текста запроса:

```bash
gemini README.md "Сделай краткое резюме"
gemini "Проверь код на ошибки" "*.js"
```

- Текстовые файлы (`.txt`, `.md`, `.json`, `.js`, `.py`, `.css` и т.д.)
- Изображения (`.jpg`, `.png`, `.gif`, `.webp`)
- Максимальный размер: 20 МБ

**Примеры с загрузкой:**
```bash
# Загрузить все JS файлы в текущей папке
gemini "Проверь код на ошибки" *.js

# Загрузить файлы из папки src рекурсивно
gemini "Сделай обзор архитектуры" "src/**/*.js"

# Анализ с автоисправлением
gemini "Исправь баги во всех python файлах" "**/*.py" --save
```

### 🔍 Примеры реальных сценариев

**Сценарий 1: Анализ без сохранения**
```bash
gemini -m flash "Найди ошибки в коде"
# Загружаете файл
# Получаете ответ и анализ
# Файлы НЕ создаются
```

**Сценарий 2: Анализ с автоисправлением**
```bash

# Загружаете файл
# Модель создает исправленный файл
```

**Сценарий 3: Dry-run (показать без сохранения)**
```bash
gemini -m flash "Создай конфиг-файл" -d
# Видите какой файл будет создан
# Файл НЕ сохраняется на диск
```

### 🎛️ Флаги для работы с файлами

| Флаг | Поведение |
|------|-----------|
| `(без флагов)` | Выводит ответ, файлы НЕ создаются |
| `--save` | Создает файлы если модель их предложит |
| `-d, --dry-run` | Показывает какие файлы будут созданы (без сохранения) |

## Решение проблем

### Ошибка: "GEMINI_API_KEY не найдена"

**Решение:** 
1. Проверьте, что файл `.env` существует в корне проекта
2. Убедитесь, что ключ скопирован правильно (без пробелов)
3. Перезагрузите терминал после добавления переменной

### Ошибка: "HTTP 429 - Too Many Requests"

**Решение:** Вы превысили лимит или квоту для этой модели. В интерактивном чате CLI предложит переключиться на другую синхронизированную модель. Также можно вручную:

```bash
/model
```

Или для одноразового запроса:

```bash
gemini -m lite "ваш запрос"
```

### Ошибка: "Module not found"

**Решение:** Переустановите зависимости:
```bash
npm install
```

### Нет вывода после запроса

**Решение:** 
- Дождитесь ответа (запрос может занять время)
- Проверьте интернет соединение
- Проверьте статус Google API в консоли

### Файлы не создаются

**Проверка:**
```bash
# Если нужны файлы - добавьте --save
gemini "создай файл" --save

# Проверьте что модель предложила сохранить
gemini "создай файл" -d  # dry-run
```

### Оптимизация запросов

```bash
gemini -m flash "Напиши функцию которая сортирует массив чисел в порядке возрастания"

# С контекстом
gemini -m flash "Я использую React. Как сделать компонент для отправки формы?"

# С явным форматом
gemini -m flash "Создай файл utils.js с 5 функциями для работы с массивами" --save
```

❌ **Не делайте:**
```bash
# Слишком расплывчато
gemini "что это"

# Без запроса на сохранение
gemini "сделай функцию"

# Не указав модель при высокой нагрузке
gemini "обработай 1000 текстов"
```

### Управление лимитами

```bash
# Проверьте перед интенсивной работой
gemini --stats

# Выберите модель с достаточным лимитом
gemini -m gemma "большой объем работы"

# Распределяйте работу между моделями
gemini -m flash "сложная задача"
gemini -m lite "простой вопрос"
```
